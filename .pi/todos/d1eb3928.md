## Implementation Summary

### CarouselScene.ts

- Added custom pan logic with space+click&drag (keeps `enablePan = false` on OrbitControls)
- Pan state: `spaceHeld`, `isPanning`, `panStartMouse/Target/CameraPos`, `panOffset`
- `onKeyDown`: detects space key, disables orbit rotation, fires `onSpaceHeldChange` callback
- `onKeyUp`: re-enables orbit rotation
- `onPanMouseDown/Move/Up`: handles drag to translate both camera and orbit target in screen-aligned world space
- Pan distance clamped to `MAX_PAN_DISTANCE = 5` from the un-panned orbit target
- Pixel-to-world mapping uses camera FOV and distance for 1:1 feel
- `resetPan()`: clears pan offset, called on slide change (`animateCameraTo`) and orbit mode exit
- `onSpaceHeldChange` callback for cursor feedback
- `dispose()` removes all keydown/keyup/mouse listeners
- `e.preventDefault()` on space key when mouse is over canvas to prevent scroll

### SimulationCanvas.svelte

- Added `spaceHeldForPan` reactive state driven by `onSpaceHeldChange` callback
- Added `pan-mode` CSS class on container for grab/grabbing cursor when space is held
- Wired up callback in the orbit controls sync effect
- Reset pan when exiting orbit mode or leaving carousel

### CarouselOverlay.svelte

- Added "Space + drag to pan" hint with move icon alongside existing orbit/zoom hints
