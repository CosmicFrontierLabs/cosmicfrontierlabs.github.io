Implemented fullscreen support for carousel explore mode using the browser Fullscreen API on `outerContainer` in `src/components/CarouselCanvas.svelte`.

### What changed
- Added fullscreen state (`isFullscreen`) and `toggleFullscreen()` handler.
- Wired `requestFullscreen()` / `exitFullscreen()` against `outerContainer`.
- Added `fullscreenchange` listener to keep Svelte state in sync (including Esc exits).
- Ensured cleanup exits fullscreen if this component unmounts while active.
- Kept render loop active in fullscreen even if IntersectionObserver reports out-of-viewport.
- Updated overlay pause logic so autoplay is not paused solely due to viewport state while fullscreen.
- Passed fullscreen props/callback into `CarouselOverlay`.
- Added a new Fullscreen / Exit fullscreen button inside explore-mode controls.
- Added fullscreen CSS rules so the carousel container reliably fills the viewport in fullscreen.

### Files modified
- `src/components/CarouselCanvas.svelte`
- `src/components/CarouselOverlay.svelte`
